gdb --pid=pid


‘thread threadno’, a command to switch among threads
‘info threads’, a command to inquire about existing threads
'frame N' - переход по функциям в стеке, где N - номер функции
'print funcname' - значение переменной
'info symbol address' - информация о функции или переменной
'break <file.cpp>:<line>' - установить точку останова
'delete breakpoints <range>' - удалить точки останова
'info breakpoints' - информация о точках останова
'break <file.cpp>:<line> if strcmp(y, "hello") == 0' - установить точку останова с условием
'info share' - информация о загруженных модулях

Полезные команды bash для отладки:

'ps uH p <PID_OF_PROCESS> | wc -l' или 'ps -L -o pid= -p <pid> | wc -l' - количество потоков в процессе
'lsof -p <PID_OF_PROCESS> | wc -l' - количество открытых файлов в процессе

Для получения номера линии по смещению(+0xd354 например) можно запустить gdb {PATH TO LIB}(lib/libautoloader.so например)
и набрать info symbol {смещение}(+0xd354 например)
